How to export alerts from SolarWinds DPA

In the DPA web console, navigate to Alerts → Alert History on the instance you want to export. Use the date range picker to select your time window, then click Export to CSV in the top-right of the alert table. The exported file includes columns: AlertName, Severity, DatabaseInstance, StartTime, AlertType, Message, Threshold, CurrentValue. Severity levels map as: Fatal → High, Critical → High, Warning → Medium, Informational → Low.

DPA-specific alert types with no Redgate Monitor equivalent: DPA Monitor Error, DPA Resource Collection Error, and Round-trip Time Probe are internal DPA health alerts that cannot be recreated — they will be flagged as Not Available in the mapping. Wait Time Anomaly and Query Throughput Anomaly alerts require Redgate Monitor v14+ Dynamic alerts for the closest equivalent.

How to export alerts from IDERA SQL Diagnostic Manager

In the SQLdm desktop client, open the Alert Browser (left panel under the monitored instance). Right-click the alert list and choose Export to CSV, or go to File → Export → Alerts. The exported file will include columns such as SQLServer, Alert, Severity, Category, Date/Time, Current Value, Threshold, and Details. You can also export from the SQLDM Repository database directly by querying SQLdmRepository.dbo.Alerts. Severity levels map as: Critical → High, Warning → Medium, Informational → Low.

How to export alerts from SQL Sentry

In the SQL Sentry client, open the Alerts log view. Arrange the columns you want (recommended: Target, Name, Type, Condition Type, Severity, Start Time, End Time, Duration, Notes), then go to File → Export Data, choose CSV, and save. The tool auto-detects SQL Sentry column names including Target (maps to database instance) and Notes (maps to message/description). Remap any column that doesn't auto-detect in step 2.

How to run the PowerShell script

1. In your Redgate Monitor portal go to Configuration → PowerShell API and download the module + generate an auth token.
2. Place RedgateSQM.psm1 in the same folder as the generated .ps1.
3. Run: .\alert_to_redgate.ps1 on any machine that can reach your Redgate Monitor server.
4. Alerts appear as timeline annotations on each matched SQL instance.
5. Alerts marked [CUSTOM METRIC REQUIRED] need a custom metric built first under Configuration → Custom metrics and alerts.
